# Vizual-X Frontend Install Manifest

## Purpose

Define exactly how the v0-generated **Vizual-X** frontend will be installed into the existing `autonomous-gpt-bridge` branch without replacing the current public site, without rebuilding the backend, and without disrupting Intelligence OS or AutoBuild OS runtime work.

## Frontend Identity

- Product/App Name: **Vizual-X**
- Brand placement: white-font `Vizual-X` wordmark in the top-left app sidebar
- Frontend benchmark: **v0.app exact benchmark path**
- Chat model layer: **Vercel AI Gateway**

## Route Strategy

Primary app route:

```text
/intelligence-os
```

Optional child routes to be installed only if generated by the v0 output and needed for organization:

```text
/intelligence-os/chat
/intelligence-os/workflow
/intelligence-os/briefs
/intelligence-os/media
/intelligence-os/investor
```

## Files Allowed to Add

The implementation may add files in these areas:

```text
app/intelligence-os/
app/api/chat/vizual-x/
components/vizual-x/
src/vizual-x/
lib/vizual-x/
```

## Files That Must Not Be Replaced

```text
app/page.js
app/page.tsx
existing public marketing pages
existing Intelligence OS and AutoBuild OS runtime files unless explicitly required for safe data binding
```

## Existing System APIs to Bind

Vizual-X must use the runtime already built on this branch. Do not create duplicate systems.

| Existing API | Vizual-X Use |
|---|---|
| `/api/workflow/autopilot-cycle` | autopilot status, next actions, exception alerts |
| `/api/intelligence/daily-command-brief` | daily command brief panel |
| `/api/intelligence/coding-autonomy-benchmark` | benchmark monitor |
| `/api/intelligence/frontier-benchmark` | frontier intelligence panel |
| `/api/intelligence/warp-speed` | simulation / upgrade status |
| `/api/agents/launch` | agent board / agent room seed |
| `/api/autobuild/status` | runtime status cards |
| `/api/release/evaluate` | release gate panel |

## Recommended Frontend Data Adapter Layer

Add a small adapter surface rather than binding every component directly to `fetch` calls.

Suggested files:

```text
src/vizual-x/api.ts
src/vizual-x/types.ts
src/vizual-x/mock-data.ts
```

Adapter responsibilities:

- centralize route URLs;
- normalize null/missing data;
- provide fallback mock data for build preview;
- keep frontend components presentation-focused;
- avoid backend drift.

## Vercel AI Gateway Chat Install Path

Frontend chat shell lives inside Vizual-X.

Server API route recommendation:

```text
app/api/chat/vizual-x/route.ts
```

Supporting files may include:

```text
src/vizual-x/chat/context.ts
src/vizual-x/chat/modes.ts
src/vizual-x/chat/prompts.ts
```

The route should:

- remain server-side;
- use Vercel AI Gateway;
- inject current Vizual-X context;
- preserve runtime safety flags;
- avoid exposing credentials;
- prepare for streaming responses.

## Quality Gate Before Install

Do not install a v0 export unless it passes:

1. Vizual-X wordmark present in the sidebar.
2. Premium v0-grade product composition.
3. Main route is `/intelligence-os`.
4. Existing homepage remains untouched.
5. All required modules are visually present.
6. Chat workspace is AI Gateway-ready.
7. Kanban workflow tracker is central.
8. Data contracts match existing APIs.
9. Empty/loading/error states exist.
10. Mobile layout remains readable.

## Integration Sequence

1. Run the split v0 prompt files as one complete prompt in v0.
2. Review the generated result only for fidelity to the manifest, not for new design invention.
3. Export or copy the v0-generated frontend code.
4. Install under `/intelligence-os`.
5. Preserve all existing backend/runtime files.
6. Add data adapters and bind existing routes.
7. Add Vercel AI Gateway chat route.
8. Verify build.
9. Verify no homepage override.
10. Verify deployment preview.
